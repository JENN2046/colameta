Implement v1.14: Stage 8 Plan Adjustment Request Preview V1.

Goal:
Add the first Stage 8 preview-only control surface for PLAN_ADJUST. This version consumes a Stage 6 CommanderDecisionRequest or equivalent validated PLAN_ADJUST context and renders a plan adjustment request/preview contract. It must not apply any change.

Required behavior:
- Add a preview-only module for plan adjustment requests.
- Accept only PLAN_ADJUST-oriented sources, such as a CommanderDecisionRequest whose requested action is ask_whether_to_prepare_plan_adjustment_draft.
- Require explicit source refs, master_taskbook_ref/hash, affected stage/version refs, proposed_change_summary, proposed_diff_or_patch_preview, and continued_master_goal_service_explanation.
- Allow optional drift_evidence_ref from Stage 7, but do not require full Stage 7 integration yet.
- Recursively reject truthy claims for apply_allowed, plan_mutated, master_goal_mutated, reviewer_bypassed, review_decision_created, gate_event_emitted, executor_continuation_authorized, commit_or_push, route_transitioned, or delivery_state_accepted.
- If a proposed change touches PROJECT_MASTER_TASKBOOK.md or master-goal fields, mark Commander hard gate required and keep apply_allowed false.
- Do not call PlanningBridge.apply_plan_patch, PlanPatchAutoApplyService, manage_plan_version apply, or any state mutation path.
- Do not add MCP/Web integration in this version.

Scope:
Allowed files are exactly the plan allowed_files. Do not touch forbidden files. This is preview-only. Do not write plan/state/taskbooks, do not create patches in .colameta/plan-patches, do not commit or push.

Validation:
Run the acceptance commands exactly. Report changed files, validation results, and residual risks. Do not commit.
