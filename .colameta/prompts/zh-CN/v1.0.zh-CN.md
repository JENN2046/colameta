## 目标

实现 ColaMeta Web 危险动作的 preview-confirm-audit 保护，覆盖 plan、patch 和
state-transition 相关 Web 动作。

`preview-confirm-audit` = 先预览、再确认、再审计。中文意思是：危险动作不能直接
执行，必须先生成可审查预览，再由确认令牌绑定，最后留下审计回执。

## 目标路由

- `/api/auto-apply-patches`
- `/api/reload-plan`
- `/api/continue-next-version`
- `/api/rerun-acceptance`
- `/api/checkpoint-review`

还要覆盖 `/api/jobs/start` 中的 `rerun_acceptance` 和 `checkpoint_review` 别名；
如果无法保护这些别名，就必须阻断它们，直到保护完成。

## 必需行为

- 缺少 confirmation 时，五个直接路由必须在 service dispatch 或 mutation 前被阻断。
- `/api/jobs/start` 的 rerun/checkpoint 别名必须在创建 job/state/thread 前被保护。
- stale state、plan、patch signature 必须被拒绝。
- 有效 confirmation 之前不能发生 mutation。
- E2A/E2B 测试继续通过。
- 不允许 executor、commit、push、preview/apply 泄漏。
- 只能返回脱敏 `dangerous_action_receipt`，不能泄漏 confirmation id。

## 保留 caveat

- E1D GitHub Actions caveat 保留。
- E2A confirmation id browser-visible caveat 保留。
- E2B 只保护 executor 前的 Web confirmation，不替代 run 后 scope validation。
- server-side E2A/E2B workdir sync 由 Commander 决策延后。
