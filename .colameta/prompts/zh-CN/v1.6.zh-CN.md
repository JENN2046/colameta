## 目标

为 v1.5 的只读 runtime/version observability surface 补充文档和测试级决策契约。

`Runtime Version Status Decision Contract` = 运行时版本状态决策契约。中文意思是：
当系统报告 `restart_needed_state` 时，人和 Agent 应该如何解释它；这个信号本身不
授权自动 restart、reload、kill 或 apply。

## 核心决策规则

### `restart_needed_state == "not_needed"`

含义：loaded runtime HEAD 和 project checkout HEAD 都已知，并且一致。

要求行为：

- 不提示 restart；
- 允许继续普通操作；
- 禁止 restart、reload、kill、service lifecycle mutation、config mutation。

### `restart_needed_state == "needed"`

含义：loaded runtime HEAD 和 project checkout HEAD 都已知，但二者不同。

要求行为：

- 只能显示 operator handoff notice；
- 可以说明运行进程可能陈旧，并提供只读证据；
- 禁止自动 restart、reload、kill、service lifecycle mutation、config mutation、
  executor workflow mutation、Git remote mutation。

### `restart_needed_state == "unknown"`

含义：loaded runtime HEAD 或 project checkout HEAD 不可用，或者无法确定。

要求行为：

- 请求或执行有边界的只读诊断；
- 说明哪个 HEAD 未知以及原因；
- 禁止自动 restart、reload、kill、apply、service lifecycle mutation；
- 禁止把 unknown 当成安全状态。

## 文档要求

文档必须说明：

- 契约目的；
- v1.5 是只读可观测性；
- not_needed、needed、unknown 三个状态的决策表；
- `restart_needed` 是信号，不是授权；
- needed 只映射到 operator handoff；
- unknown 只映射到 read-only diagnostics；
- 没有任何状态授权自动 restart、reload、kill、apply；
- 本契约不授权 Git fetch、pull、push、tag、release；
- 本契约不授权 service lifecycle mutation；
- 每个状态下 Agent 可以怎么说；
- 每个状态下 Agent 禁止做什么。
