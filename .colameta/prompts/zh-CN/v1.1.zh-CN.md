## 目标

用和 E2C 相同的危险动作 preview-confirm-audit 模型，保护 Web Console 的本地
Git commit confirmation 路由。不得修改 MCP Git manager internals，也不得增加
Web 远端 push/pull 动作。

## 范围

- 给 `/api/commit-confirm` 增加 dangerous-action confirmation。
- `/api/commit-preview` 仍然是较低风险的 runtime metadata preview，本版本不要求
  dangerous confirmation。
- 实际 commit 仍必须走 `MCPGitCommitManager.commit(preview_id)`，保留其内部
  preview、HEAD、diff、file-set guard。
- Web dangerous confirmation 必须在 `_api_commit_confirm_with_project()` 调用
  manager.commit() 前完成。
- commit-confirm 响应必须附带脱敏 dangerous receipt，不能泄漏 confirmation_id。

## 必需行为

1. `/api/commit-confirm` 缺少 dangerous confirmation 时，在调用 manager.commit()
   前返回 403。
2. expired、reused、wrong route、wrong action、wrong payload、stale HEAD、stale
   runner state、stale plan、stale commit preview signature 都必须在 commit 前拒绝。
3. 有效 dangerous confirmation 允许现有 commit confirm flow 调用一次 manager.commit()。
4. 响应包含 `dangerous_action_receipt`，且 confirmation_id 被脱敏。
5. 完整 confirmation id 不得出现在响应 payload。
6. 现有 E2C Web plan/state dangerous-action 测试继续通过。
7. commit preview/confirm 兼容现有行为，只在 commit-confirm 增加确认要求。
8. Web remote Git status 保持只读，不增加 Web push/pull apply route。

## 实现提示

- 将 `/api/commit-confirm` 加入 `DANGEROUS_WEB_CONFIRMATION_ROUTES`。
- 增加本地 Git 历史策略，例如 `action_type=git_commit_confirm`、
  `risk_class=git_local_history_action`。
- confirmation context 绑定当前 HEAD、runner state signature、plan signature 和
  commit preview signature。
- commit preview signature 应代表即将使用的 pending Web commit preview，包括
  preview_id、diff_hash、commit_files、version、project_root。
- apply 后不得打印或返回真实 confirmation id，只能出现 `REDACTED`。
- 记录 `/api/commit-preview` 是 runtime preview metadata，而 `/api/commit-confirm`
  是本地 Git history write boundary。

允许 stubbed commit confirm 并返回脱敏 receipt。文档说明 commit-preview 只是
metadata preview，commit-confirm 是需要 dangerous confirmation 的本地 Git history
mutation boundary。
