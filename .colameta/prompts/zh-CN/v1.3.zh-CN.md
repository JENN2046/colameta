## 目标

编写 docs-only RFC，定义当 ChatGPT 因平台级工具安全层阻断，无法调用 ColaMeta
preview-bound apply action 时，如何进入 operator handoff fallback protocol。

本版本不实现 handoff 系统，只写协议文档，并从 README.md 和 README.zh-CN.md
链接过去。

## 必需文件

1. 新建 `docs/platform-blocked-operator-handoff.md`。
2. 在 README.md 的 Web Console / Start ColaMeta 安全说明附近添加短引用。
3. 在 README.zh-CN.md 的对应位置添加中文短引用。

## RFC 必需内容

RFC 需要包含 status、purpose、problem、non-goals、与 Web remote Git mutation
prohibition 的关系、action-specific approved operator surfaces、platform-blocked
tool call fallback policy、OPERATOR_HANDOFF manifest schema、Action Binding Matrix、
OPERATOR_RECEIPT schema、receipt verification rules、closeout outcome taxonomy、
MVP scope、explicitly excluded actions、security invariants、future work。

## 强制策略约束

- RFC 不授权 Web Console remote Git mutation。
- `push_apply`、`fetch_apply`、`pull_apply` 不进入 Web handoff MVP。
- receipt 是 claim，不是 proof。
- closeout 必须通过只读工具验证状态。
- approved operator surfaces 必须是 action-specific。
- 不得触碰实现文件。

## 动作范围

第一版 RFC MVP 可以覆盖 `continue_next_version`、`apply_preview`、`run_once`、
`manual_validation_apply` 作为候选 handoff actions。

`commit_apply` 只记录为现有 guarded behavior，即通过当前 commit-confirm dangerous
confirmation boundary。

明确排除 `push_apply`、`fetch_apply`、`pull_apply`、`restore_file_apply`、
`revert_apply`、`tag`、`release`。

## README 要求

README 只加短指针，不复制 RFC 全文，也不能削弱 Web remote Git 只读状态规则。

## 非目标

不实现 Web Console handoff route、不实现 MCP tool、不修改 Git remote/commit/MCP
server/Web Console、不加测试、不授权 shell fallback、manual state edit、unbound apply、
skip preview、skip closeout、force push、tag、release、pull/restore/revert apply。

## 验收意图

文档要清楚说明平台阻断不是 bypass，而是 operator handoff boundary；receipt 是
未验证 claim；closeout 通过只读验证；兼容现有 Web remote Git mutation 禁止基线；
diff 只包含 RFC 和 README 指针。
