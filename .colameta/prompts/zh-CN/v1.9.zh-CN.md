## 目标

为现有 runtime/version observability surface 增加只读证明，让 operator 能判断当前运行的
MCP/Web process 是否正在服务当前 Git checkout 的代码，还是仍在使用本地编辑或 post-commit
HEAD 变化之前加载的陈旧 module source。

`Reload Awareness` = 重载感知。中文意思是：系统可以告诉你“当前运行进程可能需要重载才能反映
磁盘上的新代码”，但它不执行重载，也不授权重载。

## 重要边界

本版本不能增加 restart、reload、kill、apply、service lifecycle mutation、executor workflow
mutation、state mutation 或 Git remote mutation。它只能报告 status 和 operator-facing reload
awareness。

## 基线

- v1.8 已完成并 push。
- 当前 baseline HEAD：`50b3416443d3350a8977116d6e64e9856e9bc1f9`
- 现有 `get_runtime_version_status` 是 MCP read-only tool，必须保持 read-only。
- 现有 `restart_needed_state` decision semantics 必须保留。

## 实现要求

1. 扩展 `runner/runtime_observability.py`，加入 runtime loaded-code verification。
2. 捕获 loaded runner modules 的 import-time evidence，不使用 shell fallback、remote Git operation、
   restart/reload 或 subprocess-based mutation。
3. 比较 running process loaded runtime HEAD 和 current project checkout HEAD。
4. 比较进程捕获的 loaded runner module file fingerprints 与当前磁盘 source fingerprints。
5. 增加只读 status fields，例如：
   - `runtime_loaded_code_stale`
   - `reload_needed_for_verification`
   - `reload_awareness_reason`
   - `loaded_runtime_head`
   - `project_checkout_head`
   - `loaded_module_source_changed`
   - `changed_loaded_modules`
   - `possibly_stale_surfaces`
6. 使用保守 fail-closed 语义：
   - loaded runtime HEAD 或 project checkout HEAD unknown 时，`runtime_loaded_code_stale` 报 unknown/null，
     `reload_needed_for_verification` 报 true。
   - loaded HEAD 与 project HEAD 不同，报告 stale true 和 reload_needed true。
   - loaded module source hash 与 import-time hash 不同，报告 stale true 和 reload_needed true。
   - 所有已知 HEAD 和 loaded module fingerprints 都一致，报告 stale false 和 reload_needed false。
7. 当有 mismatch evidence 时，识别可能陈旧的 tool/result surfaces。这只是只读分类，例如 MCP tool
   results、Web Console handlers、executor workflow code paths 或 runtime observability itself。
8. 除非实现直接证明完整 Git worktree cleanliness，否则不能声称 worktree clean。若完整 dirty detection
   会扩大边界，则记录限制，并将 changed loaded source files 分类为 reload verification risk。
9. `get_runtime_version_status` 在 `runner/mcp_server.py` 中必须保持 `mcp:read`。只有为了清晰暴露扩展
   read-only data 时才可修改 `runner/mcp_server.py`。
10. 新增 `tests/test_runtime_loaded_code_verification.py`，并按需更新 runtime/MCP observability tests。
11. 新增 `docs/runtime-loaded-code-verification.md`，必要时增加 README 指针。

## 预期行为矩阵

- HEAD 都已知且相同，loaded module hashes 匹配：`runtime_loaded_code_stale=false`，
  `reload_needed_for_verification=false`。
- project checkout 有比 loaded runtime HEAD 更新的 commit：stale true，reload_needed true，
  reason 为 `loaded_head_differs_from_project_head`。
- loaded runner module 文件在 process import 后被修改：stale true，reload_needed true，
  reason 为 `loaded_module_source_changed`。
- loaded runtime HEAD unknown 或 project checkout HEAD unknown：stale unknown/null，reload_needed true，
  reason 为 `unknown_runtime_or_checkout_head`。

## 测试要求

测试必须验证：

- MCP tool registration 和 `mcp:read` scope 保持不变。
- `get_runtime_version_status` 仍返回 `ok=true`、`read_only=true`、`side_effects=false`。
- matching loaded/project HEAD 且 module fingerprints 匹配时报告 not stale。
- mismatched loaded/project HEAD 报 stale，并且 `reload_needed_for_verification=true`。
- changed loaded module fingerprint 报 stale，并识别 affected surfaces。
- unknown HEAD fail closed 到 `reload_needed_for_verification=true`，但不授权 restart/reload。
- 不暴露 restart、reload、kill、apply 字段或动作。
- 不引入 Git remote mutation、executor workflow mutation 或 service lifecycle mutation。
