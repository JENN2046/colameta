## 目标

实现一个小型只读分类规则，用来解释 executor-session head mismatch。

系统应明确区分：

1. active operation 的 recorded session HEAD 与当前 project HEAD 不同；
2. 项目已经完成并推进到新 HEAD 后，遗留的 completed idle stale session metadata；
3. 证据不足时的 unknown 或 incomplete mismatch，系统应 fail closed。

此任务不得清理 session、restart service、reload code、kill process、start executor、
commit、push 或 mutate remote state。

## completed_idle_stale_session 推荐提示

中文含义：

历史 executor session resume metadata 记录了一个与当前 project HEAD 不同的 HEAD。当前没有
operation running，job idle，Runner state 已完成。这个 warning 不表示 executor 正在运行，也
不授权 restart、reload、kill、apply、commit 或 push。新开发版本应从当前 HEAD 开始。

## 分类规则

### Rule A：无 mismatch

如果不存在 executor session，或 recorded session HEAD 与 current Git HEAD 一致：

- `status=none`
- `severity=info`
- 不增加 blocker；
- 不显示高风险 warning。

### Rule B：Active Operation Head Mismatch

如果 `operation_running=true`，或 live claim/run 仍在运行，并且 recorded session HEAD 与当前
Git HEAD 不同：

- `status=active_operation_head_mismatch`
- `severity=blocked`
- `blocks_auto_resume=true`
- `blocks_auto_start=true`
- 需要人工判断；
- 不自动修改 runtime/session files。

### Rule C：Completed Idle Stale Session

若以下条件全部成立：

- executor session 存在；
- recorded session HEAD 与 current Git HEAD 不同；
- `operation_running=false`；
- job status idle；
- latest run 或 claim completed；
- Runner state 是 `VERSION_PASSED` 且 current version 是 `PASSED`；
- worktree clean；

则：

- `status=completed_idle_stale_session`
- `severity=warning`
- `blocks_auto_resume=true`，或至少 `should_resume=false`
- `blocks_auto_start=false`
- 不显示成 active operation；
- 不要求自动 cleanup；
- 不授权 restart、reload、kill、apply、commit 或 push。

### Rule D：Unknown 或 Incomplete Evidence

如果 recorded session HEAD 与 current Git HEAD 不同，但系统无法确认 operation、job、
latest-run、state、worktree evidence：

- `status=unknown_head_mismatch`
- `severity=warning` 或 `blocked`，遵循现有 fail-closed safety conventions；
- `blocks_auto_resume=true`；
- auto-start behavior 遵循现有 Runner policy；
- `operator_message` 必须说明 evidence incomplete。

## 显示要求

至少在一个现有 status surface 暴露分类，优先：

- `/api/status`
- `/api/v2/status`
- executor continuation preview/decision
- MCP manage_executor_workflow status

不得增加新的 dangerous mutation route。

Status output 和 UI text 不能只显示原始 `head_mismatch` 标签。operator 必须能看出这是
historical stale session metadata，还是 active operation mismatch。

`completed_idle_stale_session` 不能显示为 operation currently running。

## 测试要求

增加或扩展聚焦测试，覆盖：

1. Session HEAD 等于 current HEAD 时，不产生 mismatch classification。
2. `operation_running=true` 且 head mismatch 时，产生 `active_operation_head_mismatch` 并阻止
   automatic resume/start。
3. `operation_running=false`、job idle、latest run completed、state PASSED、worktree clean 且
   head mismatch 时，产生 `completed_idle_stale_session`。
4. `completed_idle_stale_session` 不授权 restart、reload、kill、apply、commit 或 push。
5. evidence incomplete 时产生 `unknown_head_mismatch` 或等价 fail-closed state。
6. Status/API output 包含 reason 或 operator_message，不只是 raw head_mismatch warning。
7. 现有 v1.9 runtime loaded-code verification 和 validation truth-source 行为不回退。
