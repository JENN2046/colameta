## 目标

实现一个很窄的只读 runtime/version observability surface。

`Runtime Version Observability` = 运行时版本可观测性。中文意思是：只报告当前
运行进程和项目 checkout 之间的版本关系，不执行 restart/reload/kill/apply。

## 必需行为

1. 增加 pure read-only runtime observability module，报告：
   - loaded process start time，ISO-8601 格式；
   - running process 捕获的 loaded source HEAD 或 build HEAD；
   - 当前项目 checkout HEAD；
   - branch/project root；
   - restart/reload 是否需要：两边 HEAD 不同则 true，相同则 false，一边不可用则 unknown。
2. 通过现有只读 MCP/status surface 或窄命名新只读 tool 暴露 metadata。
3. 不增加 Web business route。
4. 不增加 restart/reload/kill/apply surface。
5. 不执行 Git remote mutation，不 fetch/pull/push/tag/release。
6. 不使用 shell fallback、subprocess shell execution 或直接 Git command；优先标准库读取和现有安全 helper。
7. 不暴露 secrets、tokens、env values、auth headers、raw command output。
8. 增加 focused tests，覆盖 matching HEAD、mismatched HEAD、missing/unknown HEAD、
   process start time、no restart side effects、read-only shape。

## 实现约束

runtime observability module 除了捕获模块/进程启动 metadata 到内存外，必须只读
且无副作用。surface 只能是 status/metadata，不能改变服务。若修改 MCP server
dispatch，只能加只读 tool/surface，并明确非变异。

不得触碰 Web Console、Git remote manager、executor workflow、state mutation、
commit manager 或 service lifecycle mutation 文件。

## 最终报告要求

报告修改文件、返回字段、`restart_needed` 是 boolean 还是支持 unknown、确认没有
restart/reload/kill 行为、没有 business route 或 remote Git mutation，并附验证命令结果。
