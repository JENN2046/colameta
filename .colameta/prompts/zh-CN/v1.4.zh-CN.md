## 目标

为 Platform-Blocked Operator Handoff RFC 实现第一段代码：只增加 schema 和
validation logic。不执行 operator handoff，不增加 Web 或 MCP surface，不修改现有
action dispatchers。

## 必需实现

1. 新建 `runner/operator_handoff.py`。
2. 新建 `tests/test_operator_handoff.py`。
3. 可选地在 `docs/platform-blocked-operator-handoff.md` 加短 implementation-status note，
   但不能重写 RFC。

## 实现范围

支持候选动作：

- `continue_next_version`
- `apply_preview`
- `run_once`
- `manual_validation_apply`

现有 guarded-only action：

- `commit_apply`

明确排除 action 和 alias：

- `push_apply`
- `fetch_apply`
- `pull_apply`
- `restore_file_apply`
- `revert_apply`
- `tag`
- `release`
- `force_push`
- `web_remote_git_mutation`
- `shell_fallback`
- `manual_state_edit`
- `unbound_apply`

## closeout outcome

定义与 RFC taxonomy 匹配的 closeout outcome 常量或 enum，包括
`verified_success`、`verified_noop`、`blocked_platform`、
`blocked_no_operator_surface`、`blocked_receipt_missing`、
`blocked_receipt_invalid`、`blocked_verification_unavailable`、
`blocked_verification_mismatch`、`blocked_excluded_action`。

## 校验函数

提供 pure/deterministic 函数：

- `validate_handoff_manifest(manifest: dict) -> dict`
- `validate_operator_receipt(receipt: dict, manifest: dict) -> dict`
- `is_excluded_action(action_class: str) -> bool`
- `is_supported_candidate_action(action_class: str) -> bool`
- `normalize_action_class(value: str) -> str`

receipt 必须被视为 claim，不是 proof。success 前必须声明 read-only closeout
verification。operator surface 必须 action-specific。必须拒绝 generic shell/file
editor/state editor/HTTP client、`remote_git_mutation=true`、`secrets_included=true`、
excluded actions、action mismatch、handoff_id mismatch、operator surface mismatch。

## 设计约束

validator 必须 pure、deterministic；不写文件、不跑 subprocess、不联网、不加 Web
route、不加 MCP tool、不调用 executor、不执行 Git、不改 state.json、不执行 preview/apply。
不导入 Web Console、MCP server、Git managers、executor workflow 或 project patch manager。
优先标准库。

## 必需测试

覆盖有效 handoff manifest、有效 receipt、receipt is claim not proof、push/fetch/pull
拒绝、commit_apply guarded-only、remote_git_mutation true 拒绝、shell surface 拒绝、
generic HTTP client 拒绝、action/handoff/operator surface mismatch 拒绝、缺少
read-only closeout 拒绝、secret flag 拒绝、unbound_apply 拒绝。

## 手动验收

本 slice 只增加 pure validator 和测试；不增加 Web route、MCP action、operator
execution surface、remote Git mutation surface；不触碰现有 dangerous action guard、
executor workflow、plan/version/patch apply 代码。
