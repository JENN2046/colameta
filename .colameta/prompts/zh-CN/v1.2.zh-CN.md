## 目标

把当前策略写成文档和负向测试基线：Web Console 远端 Git mutation 继续被禁止。
本版本基于 E2D commit-confirm guard，不增加任何 Web push/pull/fetch apply route。

## 必需行为

1. Web remote Git status 仍然可以作为只读状态表面读取。
2. Web Console 不得暴露 push apply、pull apply、fetch apply 或等价远端 Git
   mutation route。
3. 如果未来意外添加 Web remote mutation route，安全测试必须失败，除非有专门
   hard-gated plan。
4. `/api/commit-preview` 仍只是 runtime metadata preview。
5. `/api/commit-confirm` 仍是本地 Git history mutation boundary，并由 dangerous
   confirmation 保护。
6. 不修改 `MCPGitRemoteManager` internals。
7. 不添加 remote Git mutation Web route。

## 实现指导

- 优先更新 `README.md` 和 `README.zh-CN.md`，明确 Web remote Git mutation 被禁止，
  只允许 read-only remote status。
- 在 `tests/test_web_console_security.py` 增加负向测试，断言代表性 Web remote
  mutation 路径不存在或被拒绝，且不能通过 Web Console 调用 remote Git mutation。
- 只有在测试支持必须时，才最小修改 `runner/web_console.py`。
- 不修改 MCP Git managers。

## 代表性禁止路径

负向测试可覆盖 push/pull/fetch apply 命名变体，并应能防止未来明显的 accidental
route names。

## 边界

不得超出 Web policy baseline。不得增加 mutation 行为。
