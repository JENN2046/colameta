```yaml
  source_sha256: 69f1697a24f60e1a2305ebe1f2b1436b23204ed416d7517bfea73c83d3940c47
  translation_status: companion_draft
  authority_status: planning_reference_only
version: v1.6.1
```

## 目标

增加一个 preview-first 的 Runner state lineage reconciliation mechanism，用来把人工
受控完成的 receipt 重新绑定回 `.colameta/state.json`。

`State Lineage Reconciliation` = 状态谱系对账。中文意思是：当 Git 真实历史和
Runner 记录状态不一致时，用受控预览和受控 apply 把 Runner 状态纠正到可解释的
谱系上。

这个版本不能运行 executor，不能修改源文件，不能改 Git remote，不能改服务生命
周期，也不能改运行中进程。

## 目标场景

当前要处理的分歧是：

- v1.5 已完成并 push，commit 为 `582f047c0989e84fd62d068dede7fa6cbbd4f7fc`。
- v1.6 已完成并 push，commit 为 `8fb86aa864a9fe98512398466fa082eb59cd60ac`。
- v1.7 plan/prompt 已完成并 push，commit 为 `cd591bc0902149ad724f6d52aae29789e5fcdb54`。
- Runner state 仍显示 `current_version=v1.5`、顶层状态
  `BLOCKED_BY_ACCEPTANCE_FAILURE`、v1.5 `FAILED_BLOCKED`、v1.6 `NOT_STARTED`、
  v1.7 `NOT_STARTED`，并且 v1.5 commit metadata 错误指向 v1.7 plan/prompt commit。

## 必需能力

在 executor workflow reconciliation surface 上实现两个受控动作：

- `state_lineage_reconciliation_preview`
- `state_lineage_reconciliation_apply`

preview 必须验证 proposed reconciliation payload，并生成可审计 preview artifact。

apply 必须要求匹配的 `preview_id`，并且如果 state、HEAD 或 worktree facts 在 preview
后发生变化，就必须 fail closed。

## Preview 输入

preview 输入要支持有边界的 version binding 列表。每个 binding 包含：

- version；
- target status，通常完成版本为 `PASSED`，未来版本为 `NOT_STARTED`；
- accepted commit hash；
- accepted commit subject；
- receipt 或 manual evidence summary；
- manual reconciliation reason。

preview 还必须绑定：

- expected current HEAD；
- expected worktree clean status；
- expected branch；
- expected project root 或 project identity；
- target next runnable version。

## Fail-Closed 检查

preview 和 apply 必须在以下情况 fail closed：

- worktree dirty；
- 当前 HEAD 不匹配 expected HEAD；
- target version 不在 plan 中；
- accepted commit 不在本地 Git history；
- accepted commit subject 与输入不匹配；
- required evidence fields 缺失；
- transition 跳过未解决的早期版本，且没有显式 binding；
- payload 试图把未来版本标记为完成但缺少 evidence；
- payload 试图修改源文件、运行 executor、push、fetch、pull、tag、release、restart、
  reload 或 kill process。

## 状态行为

针对当前目标对账，preview 应能提出：

- 将 v1.5 标记为 `PASSED` 并绑定到 `582f047c0989e84fd62d068dede7fa6cbbd4f7fc`；
- 将 v1.6 标记为 `PASSED` 并绑定到 `8fb86aa864a9fe98512398466fa082eb59cd60ac`；
- 保持 v1.7 为 `NOT_STARTED`；
- 调整顶层 Runner status 和 current/next version 字段，使 v1.7 成为下一个 runnable version；
- 删除或 supersede 错误指向 `cd591bc0902149ad724f6d52aae29789e5fcdb54` 的 v1.5 commit metadata。

apply 只能触碰 `.colameta/state.json` 和受控 preview bookkeeping。不得修改 source、
docs、tests、plan、prompt、runtime implementation files、Git commits、tags、
remotes、service lifecycle state、executor sessions 或 process lifecycle。

## 审计要求

preview 和 apply 响应必须包含 before/after summary：

- old/new top-level Runner status；
- old/new current version/status；
- old/new next version/status；
- 每个 binding 的 old/new status；
- 每个 binding 的 old/new commit metadata；
- expected current HEAD；
- redacted evidence summaries；
- apply 会触碰的文件；
- warnings 和 blockers。

preview artifact 必须 one-use，并且应像现有 executor workflow preview 一样过期。

## 测试要求

聚焦测试至少覆盖：

- 为 v1.5/v1.6 完成、v1.7 未开始生成 preview；
- apply 要求 preview id；
- apply 只修改预期 state 字段；
- dirty worktree fail closed；
- HEAD mismatch fail closed；
- missing plan version fail closed；
- missing local commit fail closed；
- missing evidence fail closed；
- unresolved earlier version skip fail closed；
- 不运行 executor、不执行 Git remote operation、不 restart/reload/kill、不修改 source file。

## 禁止范围

不做 executor run、bounded loop、runtime version decision evaluation source implementation、
Web Console route changes、Git remote mutation、service lifecycle mutation、restart/reload/
kill/apply、直接手工编辑 `.colameta/state.json`、commit、push、tag 或 release。
