Implement v1.9 Runtime Loaded-Code Verification / Post-Commit Reload Awareness.

Goal:
Add read-only proof to the existing runtime/version observability surface so an operator can tell whether the currently running MCP/Web process is serving code loaded from the current Git checkout or from stale module sources after local edits or a post-commit HEAD change.

Important boundary:
This version must not add restart, reload, kill, apply, service lifecycle mutation, executor workflow mutation, state mutation, or Git remote mutation. It must only report status and operator-facing reload awareness.

Baseline:
- v1.8 is complete and pushed.
- Current baseline HEAD is 50b3416443d3350a8977116d6e64e9856e9bc1f9.
- Existing get_runtime_version_status is an MCP read-only tool and must remain read-only.
- Existing restart_needed_state decision semantics must be preserved.

Implementation requirements:
1. Extend runner/runtime_observability.py with runtime loaded-code verification.
2. Capture import-time evidence for loaded runner modules without using shell fallback, remote Git operations, restart/reload, or subprocess-based mutation.
3. Compare the running process loaded runtime HEAD against the current project checkout HEAD.
4. Compare loaded runner module file fingerprints captured by the process against current on-disk source fingerprints for loaded runner modules that are relevant to MCP/runtime observability.
5. Add read-only status fields such as:
   - runtime_loaded_code_stale
   - reload_needed_for_verification
   - reload_awareness_reason
   - loaded_runtime_head
   - project_checkout_head
   - loaded_module_source_changed
   - changed_loaded_modules
   - possibly_stale_surfaces
6. Use conservative fail-closed semantics:
   - If loaded runtime HEAD or project checkout HEAD is unknown, report runtime_loaded_code_stale as unknown/null and reload_needed_for_verification as true.
   - If loaded HEAD differs from project HEAD, report runtime_loaded_code_stale true and reload_needed_for_verification true.
   - If a loaded module source hash differs from its import-time hash, report runtime_loaded_code_stale true and reload_needed_for_verification true.
   - If all known heads and loaded module fingerprints match, report runtime_loaded_code_stale false and reload_needed_for_verification false.
7. Identify tool/result surfaces that may be stale when mismatch evidence exists. Keep this as a read-only classification, for example MCP tool results, Web Console handlers, executor workflow code paths, or runtime observability itself when corresponding loaded modules changed.
8. Do not claim complete Git worktree cleanliness unless it is directly proven by the implementation. If full dirty detection is not available without broadening the boundary, document the limitation and classify changed loaded source files as reload verification risk.
9. Keep get_runtime_version_status registered as mcp:read in runner/mcp_server.py. Modify runner/mcp_server.py only if required to expose the expanded read-only data cleanly.
10. Add focused tests in tests/test_runtime_loaded_code_verification.py and update existing runtime/MCP observability tests as needed.
11. Add documentation in docs/runtime-loaded-code-verification.md and short README pointers if useful.

Expected behavior matrix:
- heads known and equal, loaded module hashes match: runtime_loaded_code_stale false, reload_needed_for_verification false.
- project checkout has a newer commit than loaded runtime HEAD: runtime_loaded_code_stale true, reload_needed_for_verification true, reason loaded_head_differs_from_project_head.
- a loaded runner module file changed on disk after process import: runtime_loaded_code_stale true, reload_needed_for_verification true, reason loaded_module_source_changed.
- loaded runtime HEAD unknown or project checkout HEAD unknown: runtime_loaded_code_stale null/unknown, reload_needed_for_verification true, reason unknown_runtime_or_checkout_head.

Tests must verify:
- MCP tool registration and mcp:read scope remain intact.
- get_runtime_version_status still returns ok true, read_only true, and side_effects false.
- matching loaded/project HEAD and matching module fingerprints report not stale.
- mismatched loaded/project HEAD reports stale and reload_needed_for_verification true.
- changed loaded module fingerprint reports stale and identifies affected surfaces.
- unknown HEAD fails closed to reload_needed_for_verification true without authorizing restart/reload.
- no restart/reload/kill/apply fields or actions are exposed.
- no Git remote mutation, executor workflow mutation, or service lifecycle mutation is introduced.
