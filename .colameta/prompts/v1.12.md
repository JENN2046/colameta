Implement v1.12: Stage 7 Drift Evidence Schema V1.

Goal:
Add the first Stage 7 runtime contract for drift evidence. This version must define a fail-closed schema/validator for drift evidence packs, without building full packs yet and without making any semantic drift judgment.

Required behavior:
- Create a narrow Stage 7 schema module for drift evidence.
- Validate the minimum evidence fields required by the Stage 7 taskbook: drift_evidence_pack_id, master_taskbook_ref, stage_taskbook_ref, version_taskbook_ref, execution_evidence_ref, changed_files, validation_truth, scope_evidence, forbidden_files_evidence, out_of_scope_evidence, master_goal_alignment_questions, reviewer_drift_checklist, and plan_adjustment_trigger_conditions.
- Require evidence refs to be explicit objects or non-empty structured lists where appropriate.
- Fail closed when master/stage/version refs are missing, execution evidence is missing, changed files are absent, validation/scope evidence is absent, or reviewer questions/checklist are absent.
- Recursively reject truthy authority claims such as semantic_alignment_pass, no_drift_confirmed, review_decision_created, gate_event_emitted, delivery_state_accepted, plan_mutated, taskbook_rewritten, executor_continuation_authorized, commit_or_push, or stage_scope_expanded.
- Expose an authority_boundary block where every authority field remains false.
- Reuse local conventions from reviewer_drift_questions, reviewer_alignment_questions, review_feedback_schema, and commander_decision_request.
- Do not add MCP/Web integration in this version.

Scope:
Allowed files are exactly the plan allowed_files. Do not touch forbidden files. Keep this source-only / preview-only. Do not mutate plan, state, taskbooks, Delivery State, Git, stable service, executor sessions, secrets, or provider configuration.

Validation:
Run the acceptance commands exactly. Report changed files, validation results, and residual risks. Do not commit.
