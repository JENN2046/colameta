Name: Runtime Version Status Documentation + Operator Decision Contract

Goal:
Document and test the decision contract for the v1.5 read-only runtime/version observability surface.

The contract defines how humans and Agents interpret restart_needed_state without granting automatic restart/reload/kill/apply authority.

Stable baseline:
- v1.5 commit: 582f047c0989e84fd62d068dede7fa6cbbd4f7fc

- worktree should start clean
- branch: main
- remote: origin/main

Core decision rules:
1. restart_needed_state == "not_needed"
   - Meaning: loaded runtime HEAD and project checkout HEAD are both known and match.
   - Required behavior: do not prompt for restart.
   - Allowed response: continue normal operation.
   - Forbidden behavior: restart, reload, kill, service lifecycle mutation, config mutation.
2. restart_needed_state == "needed"
   - Meaning: loaded runtime HEAD and project checkout HEAD are both known and differ.
   - Required behavior: surface an operator handoff notice only.
   - Allowed response: explain that the running process may be stale and provide read-only evidence.
   - Forbidden behavior: automatic restart, reload, kill, service lifecycle mutation, config mutation, executor workflow mutation, Git remote mutation.
3. restart_needed_state == "unknown"
   - Meaning: loaded runtime HEAD or project checkout HEAD is unavailable or undetermined.
   - Required behavior: request or perform bounded read-only diagnostics.
   - Allowed response: explain which HEAD is unknown and why.
   - Forbidden behavior: automatic restart, reload, kill, apply, service lifecycle mutation, or treating unknown as safe.
