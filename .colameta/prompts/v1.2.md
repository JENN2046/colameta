E2E_WEB_REMOTE_GIT_MUTATION_POLICY_BASELINE

Goal:
Codify the current policy that Web Console remote Git mutation remains prohibited. Build a documentation and negative-test baseline from the E2D commit-confirm guard without adding any Web push/pull/fetch apply route.

Required behavior:
1. Web remote Git status remains readable as a read-only status surface.
2. Web Console must not expose push apply, pull apply, fetch apply, or equivalent remote Git mutation routes.
3. Security tests must fail if a future Web remote mutation route is accidentally added without a dedicated hard-gated plan.
4. /api/commit-preview remains runtime metadata preview only.
5. /api/commit-confirm remains the local Git history mutation boundary guarded by dangerous confirmation.
6. MCPGitRemoteManager internals are not modified.
7. No remote Git mutation Web route is added.

Implementation guidance:
- Prefer documentation updates in README.md and README.zh-CN.md that explicitly state Web remote Git mutation is prohibited and only read-only remote status is allowed.
- Add negative tests in tests/test_web_console_security.py that assert representative Web remote mutation paths are absent or rejected and cannot dispatch a remote Git mutation through Web Console.
- Only touch runner/web_console.py if a minimal explicit deny helper or test support is necessary. Do not add mutation behavior.
- Do not modify MCP Git managers.

Representative prohibited Web paths for negative testing may include push/pull/fetch apply naming variants, but the implementation should be robust to obvious future accidental route names.

Do not broaden scope beyond the Web policy baseline.
