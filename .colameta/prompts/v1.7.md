TASK: RUNTIME_VERSION_STATUS_DECISION_EVALUATOR_V1

Version:
v1.7

Name:
Runtime Version Status Decision Evaluator V1

Goal:
Add a pure function evaluator for v1.6's runtime version status decision contract.

The evaluator must accept restart_needed_state and return a structured decision object with:
- decision_kind
- operator_notice_required
- diagnostics_required
- forbidden_actions

Required behavior:
1. restart_needed_state == "not_needed"
   - decision_kind: normal_operation
   - operator_notice_required: false
   - diagnostics_required: false
   - no restart prompt
   - forbidden_actions must still include restart/reload/kill/apply and mutation actions

2. restart_needed_state == "needed"
   - decision_kind: operator_handoff_notice
   - operator_notice_required: true
   - diagnostics_required: false
   - output must make clear this is operator handoff only
   - forbidden_actions must include automatic restart/reload/kill/apply and mutation actions

3. restart_needed_state == "unknown"
   - decision_kind: bounded_read_only_diagnostics
   - operator_notice_required: false
   - diagnostics_required: true
   - output must make clear unknown is not safe and does not authorize action
   - forbidden_actions must include automatic restart/reload/kill/apply and mutation actions

4. invalid, missing, or unexpected restart_needed_state
   - fail closed as bounded_read_only_diagnostics
   - diagnostics_required: true
   - no automatic action is authorized

Implementation requirements:
- Add runner/runtime_version_decision.py.
- Add tests/test_runtime_version_decision.py.
- The evaluator must be a pure function: no file IO, no subprocess, no shell, no network, no state mutation, no Git operation, no service lifecycle operation.
- Do not modify runner/runtime_observability.py.
- Do not modify runner/mcp_server.py.
- Do not register a new MCP tool.
- Do not add a Web route.
- Do not add restart/reload/kill/apply behavior.

Suggested function name:
evaluate_runtime_version_status_decision

Suggested constant:
FORBIDDEN_RUNTIME_VERSION_STATUS_ACTIONS

Suggested forbidden actions include at least:
- restart
- reload
- kill
- apply
- service_lifecycle_mutation
- state_mutation
- config_write
- git_fetch
- git_pull
- git_push
- git_tag
- git_release
- executor_workflow_mutation
- web_route_mutation

Tests must verify:
- not_needed returns normal_operation, no operator notice, no diagnostics required.
- needed returns operator_handoff_notice and operator_notice_required true.
- unknown returns bounded_read_only_diagnostics and diagnostics_required true.
- invalid input fails closed to bounded_read_only_diagnostics.
- every decision includes forbidden_actions.
- forbidden_actions include restart/reload/kill/apply and mutation actions.
- the evaluator has no side effects and does not expose executable restart/reload/kill/apply callables.
