Implement v1.13: Stage 7 Drift Evidence Pack Builder V1.

Goal:
Build the Stage 7 drift evidence pack generator on top of the v1.12 schema. This version turns existing Stage 4-6 bounded evidence into a reviewer-ready drift evidence pack, but it must not decide whether drift exists.

Required behavior:
- Add a pack builder that composes structured drift evidence from existing audit/scope/handoff/review evidence inputs.
- Reuse v1.12 drift evidence schema validation for the output contract.
- Preserve separate sections for executor drift evidence, task drift evidence, stage drift evidence, master goal alignment questions, reviewer drift checklist, and plan adjustment trigger conditions.
- Reuse or call reviewer_alignment_questions and reviewer_drift_questions where appropriate, while keeping reviewer answer fields empty/unset.
- Include changed files, validation truth, scope evidence, forbidden files evidence, out-of-scope evidence, known gaps, and remaining risks when supplied.
- If evidence is incomplete, return a failed-closed result with blocker codes instead of fabricating no-drift or alignment success.
- PLAN_ADJUST trigger output may say that Stage 8 preview is required, but must not create a plan diff, plan patch, apply request, or taskbook mutation.
- Do not add MCP/Web integration in this version.

Scope:
Allowed files are exactly the plan allowed_files. Do not touch forbidden files. Do not write plan/state, do not create ReviewDecision or GateEvent, do not continue executor, do not commit or push.

Validation:
Run the acceptance commands exactly. Report changed files, validation results, and residual risks. Do not commit.
