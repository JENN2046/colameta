PLATFORM_BLOCKED_OPERATOR_HANDOFF_RFC

Goal:
Write a docs-only RFC that defines the fallback protocol for cases where ChatGPT cannot invoke a ColaMeta preview-bound apply action because the platform-level tool safety layer blocks the tool call before ColaMeta can evaluate its own preview, binding, and guard checks.

This version must not implement the handoff system. It only documents the protocol and links it from README.md and README.zh-CN.md.

Required files:
1. Create docs/platform-blocked-operator-handoff.md.
2. Add a short reference from README.md near the Web Console / Start ColaMeta safety notes.
3. Add the equivalent short reference from README.zh-CN.md near the Web Console / 启动 ColaMeta safety notes.

Required RFC content:
1. Status, Purpose, Problem, and Non-goals.
2. Relationship to the existing Web remote Git mutation prohibition.
3. Approved operator surfaces, explicitly action-specific.
4. Fallback policy for platform-blocked tool calls.
5. Common OPERATOR_HANDOFF manifest schema.
6. Action Binding Matrix for each supported action class.
7. OPERATOR_RECEIPT schema.
8. Receipt verification rules.
9. Closeout outcome taxonomy.
10. MVP scope.
11. Explicitly excluded actions.
12. Security invariants.
13. Future work.

Mandatory policy constraints:
1. RFC must not authorize Web Console remote Git mutation.
2. push_apply, fetch_apply, and pull_apply are excluded from the Web handoff MVP.
3. Receipt is a claim, not proof.
4. Closeout must verify state through read-only tools.
5. Approved operator surfaces must be action-specific.
6. No implementation files may be touched.

Action scope:
- The first RFC MVP may cover continue_next_version, apply_preview, run_once, and manual_validation_apply as candidate handoff actions.
- commit_apply must be documented as existing guarded behavior only through the current commit-confirm dangerous confirmation boundary.
- push_apply, fetch_apply, pull_apply, restore_file_apply, revert_apply, tag, and release are explicitly excluded from the first Web/operator handoff MVP.

Required README behavior:
- README.md and README.zh-CN.md should only add a short pointer to docs/platform-blocked-operator-handoff.md.
- Do not duplicate the full RFC in the README files.
- Do not weaken the existing statement that Web remote Git is read-only status only.

Non-goals:
- Do not implement any Web Console handoff route.
- Do not implement any MCP tool.
- Do not modify runner/mcp_git_remote.py.
- Do not modify runner/mcp_git_commit.py.
- Do not modify runner/mcp_server.py.
- Do not modify runner/web_console.py.
- Do not add tests in this RFC-only slice.
- Do not authorize shell fallback, manual state edits, unbound apply, skipping preview, skipping closeout, force push, tag, release, pull apply, restore apply, or revert apply.

Acceptance intent:
- Documentation clearly states that platform-blocked tool calls become an operator handoff boundary, not a bypass.
- The RFC preserves preview/apply auditability.
- The RFC states that receipts are untrusted claims until verified through read-only closeout.
- The RFC is compatible with the existing Web remote Git mutation prohibition baseline.
- The diff contains only docs/platform-blocked-operator-handoff.md, README.md, and README.zh-CN.md.
