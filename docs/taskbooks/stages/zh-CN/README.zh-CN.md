## 1. 阶段任务书是什么意思

`Stage Taskbook` = 阶段任务书。它把 Master Taskbook 的最高目标拆成一个阶段
内可审查、可引用、可继续细分的交付边界。

阶段任务书只拥有“阶段声明”和“阶段边界”，不拥有运行事实、不拥有验收状态、
不替代 Version Execution Taskbook，也不授权 executor 派发。

## 3. 共同边界

这些 Stage 文件都是 `discussion_draft`，中文意思是“讨论草稿”。它们不是：

- 实现授权；
- executor 运行授权；
- commit 或 push 授权；
- route transition 授权；
- delivery state 状态推进授权；
- Master Taskbook 修改授权。

## 4. 技术词汇

- `discussion_draft` = 讨论草稿。
- `planning_reference_only` = 只能作为规划参考。
- `MVP` = 最小可行闭环。
- `Stage 0-6 Thin Governed Loop` = 阶段 0-6 薄治理闭环。
